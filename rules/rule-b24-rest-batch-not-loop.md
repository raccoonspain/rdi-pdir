# REST-вызовы в цикле → `batch()`, не `call()` построчно

## Правило

Если нужно вызвать один и тот же REST-метод для N идентификаторов (задачи,
пользователи, элементы CRM и т.п.) — использовать `$b24->batch([...])`
(до 50 команд за раз, при бо́льшем количестве — `array_chunk` по 50), а не
`foreach ($ids as $id) { $b24->call(...); }`.

```php
// Плохо — N последовательных round-trip'ов
foreach ($taskIds as $tid) {
    $res = $b24->call('tasks.task.get', ['taskId' => $tid]);
}

// Хорошо — 1 round-trip на каждые 50 id
foreach (array_chunk($taskIds, 50) as $chunk) {
    $commands = [];
    foreach ($chunk as $tid) {
        $commands[(string)$tid] = ['tasks.task.get', ['taskId' => $tid]];
    }
    $res = $b24->batch($commands);
    // результат конкретной команды: $res['result']['result'][$tid]
    // ошибка конкретной команды:    $res['result']['result_error'][$tid]
}
```

## Почему

Каждый `call()` — отдельный HTTPS round-trip к Б24 (~100-150мс на стороне
Б24, не считая сетевой задержки до сервера). При N=50-100 (например,
ключевая задача на каждый модуль в доске проекта) последовательные вызовы
складываются в 10-15+ секунд ожидания — воспринимается пользователем как
«приложение зависло», хотя сам код формально корректен и оптимизации
выглядят неочевидными (каждый вызов сам по себе быстрый).

`batch()` уже есть в `b24.php` (`B24->batch()`), но код по умолчанию его не
использует — писать `foreach` с `call()` внутри проще и работает на малых
N, грабля не видна на тестовых данных с парой записей.

## Как поймать до продакшна

Не судить по количеству строк кода/сложности — судить по числу REST-вызовов
на один HTTP-запрос эндпоинта. Если в эндпоинте `foreach` вокруг `$b24->call()`
(не `batch()`) — это кандидат на батчинг, независимо от того, сколько записей
в тестовых данных на момент разработки.

## Практика

`ap-pp`, `api/board.php` (доска проекта, вкладка «Пульт проекта»): цикл по
`tasks.task.get` на ключевую задачу каждого модуля/этапа — при ~85 модулях
загрузка занимала 24-34 секунды. После перевода на `batch()` (чанки по 50,
см. `fetchTasksByIds()` в `board.php`) — 2-3 секунды. Сетевая часть тут была
ни при чём (замерено и на старом, и на новом сервере — разница та же).
