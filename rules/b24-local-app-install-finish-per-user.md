# installFinish отслеживается per-user, а не одним флагом

## Контекст

Официальный флаг `INSTALLED` в Б24 — портальный, не per-user
(`app.info().INSTALLED`, один на весь портал). Но `www/index.php` в этом
шаблоне обслуживает install-путь и runtime одним URL, и разным пользователям,
открывающим приложение, нужно показать корректную страницу именно им —
не полагаясь только на портальный флаг.

## Решение

`B24::markUserFinished($userId)` / `B24::needsInstallFinishFor($userId)` в
`www/api/b24.php` держат список `installFinishedUsers[]` внутри
`data/b24-tokens.php`. Логика:

- Список пуст → `needsInstallFinishFor()` вернёт `true` для любого юзера
  (свежая установка, ещё никто install-finish не проходил).
- `_unknown` в списке → все дальнейшие юзеры считаются завершившими установку
  (см. `resolveUserIdFromAuth()` — если ID юзера не резолвится, помечаем
  `_unknown`, чтобы не зациклить показ install-finish навсегда).
- Конкретный `userId` в списке → для него install-finish уже не нужен.

## Почему не один общий булев флаг

Два администратора могут открыть приложение независимо (например, при
демонстрации коллеге). Общий флаг `installed: true/false` либо покажет
install-finish страницу второму админу без необходимости (если флаг
привязывать к «первому» открытию не так), либо, что хуже, не покажет её вообще
никому кроме самого первого — а `BX24.installFinish()` должен быть вызван
именно из фрейма конкретной сессии пользователя, который сейчас смотрит на
install-путь. Per-user список — самый простой способ не потерять ни один
из этих случаев.
