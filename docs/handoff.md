# ПЕРЕДАЧА ПРОЕКТА — ЧИТАЙ ЭТО ПЕРВЫМ

> Этот файл — точка входа для **новой нейронки** (или человека), которая
> впервые открывает проект. Цель: за 5 минут понять **что это, где мы
> сейчас и куда идём** — без необходимости читать весь код.

---

## ⚠️ Это копия проекта `ap-pdir`, не проект с нуля

Тиражирование под второго клиента на том же движке «Пульта» (иерархия
Сделка → Этап → Модуль в Б24) — портал и ID бизнес-процессов/полей у
нового клиента другие. Перенесено как есть: код `www/`, инженерные
грабли `rules/`. **Не перенесено** (архивировано отдельно, не потеряно):

- Бизнес-контекст сущностей CRM **первого** клиента (Альфа) —
  `/source/prev-project/` (АП Deal/Milestone/Module/Pay/Задачи).
- Журнал и решения первого проекта — `docs/prev-project/`
  (`project-brief.md`, `state.md`, `changelog.md`, `decisions.md`,
  `superpowers/` — спеки и планы фич).

**Нужно от разработчика, прежде чем писать бизнес-логику под нового
клиента:** актуальная информация по сущностям Б24 нового портала —
аналог `source/prev-project/АП *.md`, но с реальными ID/UF-полями нового
клиента (не полагаться на аналогию со старым порталом — коды кастомных
полей Б24 генерирует независимо для каждого портала, см. `docs/state.md`).

## Как устроена память проекта

Весь контекст живёт в четырёх файлах. Читай их **в этом порядке**:

| # | Файл | Отвечает на вопрос | Как меняется |
|---|------|--------------------|--------------|
| 1 | [project-brief.md](./project-brief.md) | **Что** это за проект и **зачем** | Редко |
| 2 | [state.md](./state.md) | **Где мы сейчас** и **что делать дальше** | Постоянно — «живой снимок» |
| 3 | [changelog.md](./changelog.md) | **Что уже сделали** (по датам) | Только дописывается вниз |
| 4 | [decisions.md](./decisions.md) | **Почему** так, а не иначе | Только дописывается вниз |

Плюс [../CLAUDE.md](../CLAUDE.md) — инструкции по стеку, деплою и правилам работы.

---

## Быстрый старт для входящей нейронки

1. Прочитай 4 файла выше по порядку.
2. Глянь последние коммиты:
   ```bash
   git log --oneline -20
   ```
3. Проверь что прод живой — открой URL из раздела «Карта проекта» в `state.md`.
4. Возьми первый пункт из «Следующие шаги» в `state.md` — и продолжай.
