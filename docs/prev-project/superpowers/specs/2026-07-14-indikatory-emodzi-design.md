# Индикаторы срыва графика/оплаты — эмодзи вместо точек

**Дата:** 2026-07-14 · **Статус:** утверждено пользователем

## Контекст

Пункт бэклога из `docs/state.md` (фидбек 2026-07-13): цветные точки
🔴🟡 в колонке «Индикаторы» мелкие, жёлтая плохо отличима от зернистой
текстуры фона таблицы. Пользователь выбрал (2026-07-14): **заменить**
точки эмодзи целиком, не добавлять эмодзи рядом с точками — символ сам
по себе несёт смысл, цвет для различения не нужен.

## Решение

`indicatorDots()` (`www/js/app.js`) вместо
`<span class="indicator-dot red/yellow">` рендерит текстовые символы:
- `broken_schedule` → ⏰
- `awaiting_payment` → 💰

`title`-атрибут (текущий текст подсказки — «Срыв графика»/«Ожидание
оплаты») сохраняется без изменений. `.indicator-dot`/`.indicator-dot.red`/
`.indicator-dot.yellow` в `www/css/style.css` удаляются — больше нет
потребителя.

## Проверка

- Высота строки таблицы не увеличивается (эмодзи в потоке текста ячейки,
  без дополнительных отступов/размеров, в отличие от `.indicator-dot`
  с фиксированной `width`/`height`) — проверить глазами после деплоя.
- Рядом с уже цветными бейджами стадий не задваивает смысл цвета — эмодзи
  не несёт цветовой семантики вовсе, конфликта нет по построению.

## Не входит в объём

- Индикаторы других уровней (Milestone/Module) не имеют такой колонки —
  не создаётся.
- Легенда/подсказка про то, что означают эмодзи — уже есть `title` на
  каждом символе, отдельной легенды не нужно (та же логика, что и у
  цветных плашек счётчиков в D-011).
