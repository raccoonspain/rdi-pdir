# Выравнивание колонок аккордеона (Этапы/Модули)

**Дата:** 2026-07-14 · **Статус:** утверждено пользователем

## Контекст

Фидбек пользователя (скриншот 2026-07-14): при раскрытии модулей у
разных этапов колонка «Стадия» (и другие) скачет по горизонтали между
блоками — визуальная каша. Причина: каждая `<table class="sub-table">`
(что для этапов, что для модулей) — отдельный DOM-элемент с
`table-layout: auto` (браузерное поведение по умолчанию), ширина
колонок считается по содержимому каждой таблицы независимо. Плюс у
каждого уровня вложенности свой намеренный отступ слева (`.detail-wrap`
— 46px, `.module-wrap` — 34px, часть визуального языка D-008,
показывает глубину вложенности) — это НЕ баг, трогать не нужно.

## Решение

Пользователь выбрал: выравнивать колонки **только внутри своего
уровня** — все под-таблицы Этапов между собой одинаковые по ширине
колонок, все под-таблицы Модулей между собой одинаковые, но между
уровнями Этапов и Модулей ширина может отличаться (отступ между ними
и так намеренно разный).

Реализация: `table-layout: fixed` + фиксированные `width` в `%` по
номеру колонки, раздельно для двух форм таблицы (у Этапов 6 колонок, у
Модулей 5, разный состав). Различаются через модификатор класса —
`sub-table--milestones` / `sub-table--modules` — добавляется в
`www/js/app.js` рядом с уже существующим `class="sub-table"`.

Ширины (сумма 100% в каждой форме):

**Этапы** (Номер/Название/Стадия/Цена/Дни КП-План/Активность):
`8% / 24% / 14% / 13% / 12% / 29%`

**Модули** (Номер/Название/Стадия/Разработчик/Активность):
`8% / 26% / 14% / 18% / 34%`

## Проверка

Глазами в браузере после деплоя: раскрыть модули у двух и более разных
этапов одной сделки (и/или разных сделок) — колонка «Стадия» модулей
должна быть на одной вертикальной линии между всеми раскрытыми блоками
модулей; то же для колонки «Стадия» этапов между разными сделками.
Отступ между уровнем Этапов и уровнем Модулей остаётся как есть
(намеренный, D-008).
