description: Use when implementing a new feature or bugfix in this Bitrix24 app, before writing implementation code — write a failing check first (REST call, page request, or test script), then make it pass.
---

Перед тем как писать реализацию:

1. Сформулируй, как ты проверишь, что фича работает — конкретный REST-вызов, конкретная
   страница/кнопка, конкретный вход/выход.
2. Прогони эту проверку ДО правки кода — убедись, что она действительно падает (а не ты
   выдумал проверку, которая и так проходит).
3. Пиши минимальный код, чтобы проверка прошла.
4. Прогони ещё раз, покажи реальный вывод — не заявляй «готово» на основании чтения кода
   глазами.

Для этого проекта «проверка» чаще всего значит: реальный запрос к `$b24->call(...)` или
открытие страницы приложения в браузере — не юнит-тест в вакууме.
