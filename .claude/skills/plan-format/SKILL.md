---
name: plan-format
description: Use in Plan agent when producing an implementation plan for this project — tie it to the existing docs/state.md journal instead of a throwaway format.
---

План должен:

- ссылаться на актуальные «Следующие шаги» из `docs/state.md`, а не изобретать структуру
  с нуля;
- явно называть файлы, которые будут созданы/изменены;
- отдельно отмечать шаги, которые требуют неочевидного решения — они после реализации
  уйдут в `/decision`;
- не включать работу, не относящуюся к текущей задаче (без «заодно порефакторю X»).
