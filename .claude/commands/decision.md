Зафиксируй принятое решение в `docs/decisions.md`, чтобы будущий исполнитель
понимал «почему так, а не иначе» и не переоткрывал закрытый вопрос.

Суть решения: $ARGUMENTS

Сделай:

1. Открой `docs/decisions.md`, найди последний номер `D-NNN`, возьми следующий.

2. Добавь запись ВНИЗ файла (после строки-маркера) по шаблону из файла:
   - **Контекст** — какую задачу решали;
   - **Варианты** — что рассматривали;
   - **Решение** — что выбрали;
   - **Почему** — главная причина + чем пожертвовали;
   - **Последствия** — на что влияет дальше.
   Если решение отменяет прежнее — поставь статус «отменяет D-XXX».

3. Если уместно — упомяни D-NNN в свежей записи `docs/changelog.md`.

4. Снимок в git и отправка на GitHub:
   ```bash
   bash scripts/snapshot.sh "решение: <короткое название> (D-NNN)"
   ```
   Скрипт сам сделает коммит и отправит на GitHub (если репозиторий подключён).

5. Ответь: какой номер присвоил и в двух словах суть.
