# Что делаем с сущностью Этапы (Milestone)

- 0️⃣ Заводим этапы из под сделки
	- 1️⃣🅰 Пишем номер этапа и название
	- 🆔 Этап привязывается к сделке
- 1️⃣ В заведенный этап пишем
	- 📅 Сроки по договору
	- 💰 Цена этапа (*без млн*)
- 2️⃣✏️Вручную дальше работаем только с полем `Ввод активности`
	- Пишем в поле `Ввод активности`
	- 🤖Переносит по полям
		- `Последняя активность`, `Дата активности`, `Процесс этапа` (дозапись)
		- Очищает  `Ввод активности`
- 🤖 Автоматические пересчеты:
	- 💰 Оплачено и остатки в сделку и в этап, через смарт оплат (Pay)
	- 💰 Оплачено и остатки в сделку и в этап при каждом обновлении Milestone 
		- $ чтобы решить проблему расчета остатков, когда не было ни одного платежа и без авансов
	- 📅 `Д План Завершения Этапа` и `Дни КП-План` через смарт Модули (Module)

# 🤖📜 Описание бизнес-процессов

## 🚀 A_Mst Создание

- $ Записи в документе Этапы
	- 🅰🔄 Переименование
		- **Извлекаем** из смарта `Сделки` уже записанные `Код объекта`
		- `Название` = `{=Deal_Obj_Code:UF_CRM_8_O_CODE}-{{Номер}}-{{Краткое наименование}}`

- $ Запись в документ Сделка
	- 🆔➕🔄 Дозапись ID смарта в Deal
		- Проблема в том, что при записи в другой смарт нет возможности дописывать во множественное поле, происходит перезатирание поэтому:
			- В переменную `AddSmartID` пишем `ID` созданного модуля
				- ! Переменная `множественная`, иначе merge не сработает
			- **Извлекаем** из смарта `Сделки` уже записанные `ID Смарт этапа`
			- Обновляем смарт `Сделки` через функцию `merge`
			- `{{= merge({=Deal_ID_Milestone:UF_CRM_8_O_MST_ID}, {=Variable:AddSmartID})  }}`



## ✏️ A_Mst Изменение

- $ Записи в документе Этапы
	- 🅰🔄 Переименование
		- **Извлекаем** из смарта `Сделки` уже записанные `Код объекта`
		- `Название` = `{=Deal_Obj_Code:UF_CRM_8_O_CODE}-{{Номер}}-{{Краткое наименование}}`
	- 🧮 🔄 Цена Этапа млн
		- Пишем в этап `{{Цена Этапа}}/1000000`
	- 🧮 🔄 Перебор ВСЕХ ОПЛАТ Этапа, *чтобы пересчитать остатки*, если мы изменили цену Этапа, а оплат не было. В противном случае, остатки будут не корректны
		- **Перебираем** ID смарт ОПЛАТ в `PAY_FACT`
		- Накапливаем сумму в `intSumPay` = `{=Variable:intSumPay}+{=PAY_FACT:UF_CRM_12_PAY_FACT}`
		- Пишем в этап Сумму оплаты `intSumPay`, `intSumPay/1000000` и остатки, как `{{Цена Этапа}}-{=Variable:intSumPay}` и `({{Цена Этапа}}-{=Variable:intSumPay})/1000000`
	- 🧮= 0️⃣ Пишем нулевую оплату если оплат не было
		- По условию `ID Смарт оплаты <> не заполнено` 
			- `Оплачено` / `Оплачено млн` = 0
			- `Остатки` / `Остатки млн` = `{{Цена Этапа}}` / `{{Цена Этапа млн}}`
	- 📜🔄 Обновление Активности Этапа
		- Проверяем заполенность поля `Ввод активности`
			- Если ЗАПОЛНЕНО, то
				- Обновляем `Последняя активность` и `Дата активности`
				- Дописываем `Процесс этапа`
				- Очищаем `Ввод активности` 
	- 📆 Дни КП-План, если нет вложенных модулей - Сравниваем Дату начала этапа с СЕГОДНЯ. Это и будет показателем KPI
		- Проверяем, что поле `ID Смарт модуля` = `НЕ ЗАПОЛНЕНО`
			- Сравниваем, что больше `Д Начала Этапа` или `СЕГОДНЯ`
				- В зависимости от этого назначаем в переменную `intMinusOne` = `-1` или `1`
			- Считаем дни и пишем в `Дни КП-План`
				- `Дни КП-План` = `{{= (datediff({{Д Начала Этапа}}, {=System:Date} , '%a')+1)*{=Variable:intMinusOne}   }}` 




- $ Запись в документ Сделка


- 1️⃣🔄 *Запись в сделку* данных на основании перебора ВСЕХ этапов связанных со сделкой
	- Подготовка накопительных и сравниваемых переменных
		- `dMin` = `{=System:Date}`
		- `dMax` = `{=System:Date}`
		- `intSumCost` = `0`
	- **Извлекаем** ID Смартов этапов в `Deal_MST_ID`
	- Запускаем перебор ID Смартов этапов (`Deal_MST_ID`) в `MST_Current_ID`. Фильтр по полю для извлечения данных из смарта `ID равно {=MST_Current_ID:Value}`
		- **Извлекаем** Дату начала этапа в `MST_DSTART`, Дату завершения этапа в `MST_DFINISH`
			- Сравниваем извлеченные даты с переменными `dMin` и `dMax`
			- Если `dMin` > Д Начала Этапа, то `dMin` = Д Начала Этапа
			- Если `dMax` < Д Завершения Этапа, то `dMax` = Д Завершения Этапа
			- ! Добавить условие, что поля Д Начала Этапа и Д Завершения Этапа Заполнены
		- **Извлекаем** Цену этапа в `MST_COST`			
		- **Извлекаем** Оплачено по этапу в `MST_PAY` и Остатки по этапу в `MST_BALANCE`
		- **Извлекаем** Остатки по этапу в `MST_BALANCE`
		- Пишем разом
			- `intSumCost` = `{=Variable:intSumCost} + {=MST_COST:UF_CRM_10_MST_COST}`
			- `intSumPay` = `{=Variable:intSumPay} + {=MST_PAY:UF_CRM_10_PAY}`
			- `intSumBalance` = `{=Variable:intSumBalance} + {=MST_BALANCE:UF_CRM_10_BALANCE}`
	- После перебора пишем в Сделку:
		- `Д Начала ДГ` = `{=Variable:dMin}`
		- `Д Завершения ДГ` = `{=Variable:dMax}`
		- `Цена контракта` = `{=Variable:intSumCost}`
		- `Оплачено по контракту` = `{=Variable:intSumPay}`
		- `Остатки по контракту` = `{=Variable:intSumBalance}`
		- `Цена контракта млн` = `{=Variable:intSumCost}/1000000`
		- `Оплачено по контракту млн` = `{=Variable:intSumPay}/1000000`
		- `Остатки по контракту млн` = `{=Variable:intSumBalance}/1000000`
