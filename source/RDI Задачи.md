# Инструменты управления

- $ Чтобы не ставить левые приложения, было принято решение управлять полями задач через вебхуки

- ❓ Список существующих полей
	- `task.item.userfield.getlist`

- $ Создание полей через конструирование запроса в окне вебхука:
- 🛠️ Метод = `task.item.userfield.add`
- Параметры
	- 📌 Тип поля
		- `PARAMS[USER_TYPE_ID]` = `string`
	- 🆔 Имя поля
		- Называем всегда с `UF_`
		- `PARAMS[FIELD_NAME]` = `UF_TASK_DEV_COMP`
	- 🅰 Название поля
		- `PARAMS[LABEL]` = `Разработчик Компания`
	- 🅰 Название поля на русском
		- `PARAMS[EDIT_FORM_LABEL][ru]` = `Разработчик Компания`
	- 🔢 Множественность
		- `PARAMS[MULTIPLE]` = `N` (или `Y`)
	- ❗ Обязательное заполнение
		- `PARAMS[MANDATORY]` = `N` (или `Y`)
	- 🔽 Сортировка
		- `PARAMS[SORT]` = `200`

- 📌 Типы полей для задач
	- `string`
	- `double`
	- `datetime`
	- `boolean`


# Пользовательские поля



| S   | Название поля             | Имя поля             | Тип поля | Множ  | Сорт |
| --- | ------------------------- | -------------------- | -------- | ----- | ---- |
| 🏭  | **Разработчик Компания**  | `UF_TASK_DEV_COMP`   | string   | N     | 200  |
| 📓  | **Разработчик Контакт**   | `UF_TASK_DEV_CONT`   | string   | **Y** | 210  |
| 📌  | **Объект наименование**   | `UF_TASK_OBJ_NAME`   | string   | N     | 220  |
| 📂  | **Объектовая папка ALL**  | `UF_TASK_OBJ_FOLDER` | string   | N     | 230  |
| 🔗  | **Внутренняя ссылка ALL** | `UF_TASK_OBJ_URL`    | string   | N     | 240  |

- 📌Особенности записи в поля при создании задачи
	- `Разработчик компания` - строковое поле, поэтому при назначении из под смарта, нужно выбирать текстовую привязку к Компании
		- Например `{{Разработчик компания (текст)}}`
	- `Разработчик контакт` - строковое поле, поэтому при назначении из под смарта, нужно выбирать текстовую привязку к Контакту + **ДОПОЛНИТЕЛЬНО** это еще и множественное поле, поэтому еще нужно взрывать массиы
		- `{{=implode(', ',{{Разработчик контакт (текст)}})}}`
	- `Объект наименование`, `Объектовая папка ALL`, `Внутренняя ссылка ALL`
		- Без особенностей, кроме того, что тянем не из под смарта модуля, а напрямую из смарта сделки, например
			- `{=OBJ_SName:UF_CRM_13_O_SNAME}`
