# Как привязать приложение к порталу Битрикс24

> Эта инструкция — для **99% случаев**: деплой на VPS `rub24.blackboxbegin.space`
> (`45.91.55.178`, по SSH), где домен и путь
> известны заранее, и `deploy.sh` сам генерирует рабочий `env.php`. Если
> деплоите на хостинг заказчика, где путь/домен заранее не известны
> (shared-хостинг, FTP-заливка) — см. [how-to-link-ALL.md](how-to-link-ALL.md)
> вместо этого файла.

## Что такое client_id и client_secret

Битрикс24 использует OAuth-авторизацию. Когда пользователь открывает приложение, Б24 выдаёт временный код, который приложение обменивает на `access_token` для REST-запросов. Для этого обмена нужны:

| Параметр | Назначение |
|---|---|
| `client_id` | Идентификатор приложения — говорит Б24, *какое* приложение делает запрос |
| `client_secret` | Секретный ключ — доказывает, что запрос делает *настоящий* владелец приложения |

Хранятся в `env.php` на сервере rub24 (`/var/www/b24/rdi-pdir/env.php`,
доступен только по SSH). **Никогда не коммитить в git.**

## Порядок действий

### 1. Задеплоить приложение

```bash
bash deploy.sh
```

При первом деплое скрипт по SSH создаёт `/var/www/b24/rdi-pdir/env.php` на
rub24 из `env.example` с уже верным `APP_URL`
(`https://rub24.blackboxbegin.space/rdi-pdir`), `APP_PATH` и `DATA_ROOT` — все
три детерминированы конвентом, править их руками не нужно.

### 2. Зарегистрировать приложение в Б24

В портале Битрикс24 нового клиента: **Приложения → Разработчикам →
вкладка «Готовые сценарии» → Другое → Локальное приложение**

- Тип: **Серверное**
- **Путь обработчика:** `https://rub24.blackboxbegin.space/rdi-pdir/index.php`
- **Путь установки:** тот же URL (с `/index.php` на конце — обязательно)
- Права (минимум): `crm`, `user`, `placement`

После сохранения Б24 покажет `client_id` и `client_secret`.

### 3. Вписать credentials в env.php

```php
define('B24_CLIENT_ID', 'app.xxxxxxxxx.xxxxxxxx');    // из карточки Б24
define('B24_CLIENT_SECRET', 'xxxxxxxxxxxxxxxxxxxxxxxx');  // из карточки Б24
```

### 4. Установить приложение

Открыть в браузере страницу приложения в Б24 или перейти по `index.php` напрямую.
При успешной установке OAuth-токены сохранятся в `data/b24-tokens.php`.

## Несколько порталов

Этот проект — **single-tenant**: один экземпляр = один портал. Другой
портал — отдельный проект/деплой, вне скоупа этой инструкции.
