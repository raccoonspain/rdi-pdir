# AGENTS.md — для Kilo Code (и совместимых агентов)

Этот файл читается автоматически Kilo Code. Claude Code его не использует напрямую — но
описанный ниже журнал и остальные правила совпадают с тем, что уже требует CLAUDE.md от
Claude Code, чтобы оба агента работали по одним и тем же правилам.

## Сначала — прочитай CLAUDE.md целиком

[CLAUDE.md](CLAUDE.md) — это не Claude-специфичный файл, а полная спецификация проекта:
стек, деплой (`deploy.sh`), инфраструктура (Caddy + PHP-FPM), REST-конвенции Б24, цепочка
поиска документации, раскладка на сервере. Обращение «Клоду» в тексте — по историческим
причинам, протокол одинаков для любого агента, включая Kilo.

**Перед первым действием в сессии — открой и прочитай CLAUDE.md целиком.** Если файла
`.onboarding-done` нет в корне — следуй протоколу онбординга (`docs/onboarding.md`), не
пропускай этот шаг из-за того, что раздел адресован «Клоду».

## Журнал проекта — обязательно перед началом и после шага

Прочитай перед началом работы:
- `docs/handoff.md` — с чего начать
- `docs/state.md` — где сейчас, следующие шаги

После осмысленного шага (фича, багфикс, деплой, решение):
1. Обнови `docs/state.md` (перепиши «Где мы сейчас», «Сейчас в работе», «Следующие шаги» — не копи старое).
2. Допиши запись в `docs/changelog.md` (сверху, под маркером).
3. Если решение неочевидное — запись в `docs/decisions.md` (D-NNN).
4. Коммит: `bash scripts/snapshot.sh "<короткое описание>"`.

Это тот же журнал, которым пользуется Claude Code в этом проекте — оба агента должны
читать и писать одни и те же файлы, а не заводить отдельную память.

## Грабли Битрикс24 — не грузи заранее, грепай по требованию

В `rules/` лежит справочник технических грабель Б24 (это не журнал!). Перед тем как
трогать install-flow, OAuth, CRM REST-вызовы — сначала:

```
grep -i "<тема>" rules/INDEX.md
```

и только потом открывай конкретный файл, если он релевантен. Не читай все файлы `rules/`
целиком «про запас» — это трата контекста.

## Прежде чем заявить, что готово

Не говори «сделано» / «работает», пока не прогнал реальную проверку (тест, запрос к API,
скриншот в браузере) и не увидел результат своими глазами.
